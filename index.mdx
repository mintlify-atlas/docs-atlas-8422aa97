genRojoTree is a Node.js tool for Roblox developers using [Rojo](https://rojo.space). Instead of hand-editing `default.project.json` every time you add a new script, genRojoTree reads your `src/` folder and generates the file automatically — keeping your Rojo project tree in sync with your filesystem.

## What genRojoTree does

When you run `npm run build:rojo`, genRojoTree walks every `.luau` file under `src/`, determines where it belongs in the Roblox service tree, and writes a fresh `default.project.json`. You can also run `npm run watch:rojo` to regenerate automatically whenever files change.

<Steps>
  <Step title="Install dependencies">
    Clone the repository and run `npm install` to install `chokidar-cli`.
  </Step>
  <Step title="Organize your source files">
    Place Luau scripts under `src/services/`, `src/classes/`, or `src/modules/` following the naming conventions.
  </Step>
  <Step title="Generate the project file">
    Run `npm run build:rojo` to produce `default.project.json` from your source tree.
  </Step>
  <Step title="Serve with Rojo">
    Run `rojo serve` to sync your project into Roblox Studio.
  </Step>
</Steps>
