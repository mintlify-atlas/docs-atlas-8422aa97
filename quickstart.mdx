---
title: "Get started with genRojoTree from clone to rojo serve"
description: "Clone the repo, install Node and Rojo dependencies, run the generator, and start syncing your Roblox project with Rojo in one workflow."
---

This guide walks you through setting up genRojoTree from scratch. By the end you will have generated a `default.project.json` from your source tree and started a live Rojo sync session with Roblox Studio.

<Steps>

  <Step title="Clone the repository">
    Clone the genRojoTree repository and open it in your terminal.

    ```bash
    git clone https://github.com/leifstout/genRojoTree.git
    cd genRojoTree
    ```
  </Step>

  <Step title="Install Node.js dependencies">
    genRojoTree uses [`chokidar-cli`](https://github.com/open-cli-tools/chokidar-cli) for watch mode. Install it with npm.

    ```bash
    npm install
    ```

    This installs the single dev dependency listed in `package.json`:

    ```json package.json
    {
      "devDependencies": {
        "chokidar-cli": "^3.0.0"
      }
    }
    ```
  </Step>

  <Step title="Install Rojo via Rokit">
    The project uses [Rokit](https://github.com/rojo-rbx/rokit) to manage the Roblox toolchain. Run `rokit install` to pull down the pinned versions of Rojo and Wally.

    ```bash
    rokit install
    ```

    The pinned versions from `rokit.toml`:

    ```toml rokit.toml
    [tools]
    rojo = "rojo-rbx/rojo@7.5.1"
    wally = "UpliftGames/wally@0.3.2"
    ```

    <Info>
      If you do not have Rokit installed, follow the [Rokit setup guide](https://github.com/rojo-rbx/rokit) before continuing.
    </Info>
  </Step>

  <Step title="Generate the Rojo project file">
    Run the build script to walk your `src/` folder and generate `default.project.json`.

    ```bash
    npm run build:rojo
    ```

    Under the hood this runs:

    ```bash
    node tools/genRojoTree.js
    ```

    You should see:

    ```
    ✅ default.project.json generated.
    ```
  </Step>

  <Step title="Inspect the generated file">
    Open `default.project.json` to verify the output. With the example `src/services/ExampleService/` layout included in the repo, it will look like this:

    ```json default.project.json
    {
      "name": "genrojotree",
      "tree": {
        "$className": "DataModel",
        "ReplicatedStorage": {
          "Shared": {
            "$className": "Folder",
            "Services": {
              "$className": "Folder",
              "ExampleService": {
                "$className": "Folder",
                "ExampleServiceClient": {
                  "$path": "src/services/ExampleService/Client.luau"
                },
                "ExampleServiceUtils": {
                  "$path": "src/services/ExampleService/Utils.luau"
                }
              }
            },
            "Classes": { "$className": "Folder" },
            "Modules": { "$className": "Folder" }
          },
          "Packages": { "$path": "Packages" },
          "UI": { "$path": "src/ui" }
        },
        "ServerScriptService": {
          "Server": { "$path": "src/startup/Server.server.luau" },
          "Services": {
            "$className": "Folder",
            "ExampleService": {
              "$className": "Folder",
              "ExampleServiceServer": {
                "$path": "src/services/ExampleService/Server.luau"
              }
            }
          },
          "Classes": { "$className": "Folder" },
          "Modules": { "$className": "Folder" }
        },
        "StarterPlayer": {
          "StarterPlayerScripts": {
            "Client": { "$path": "src/startup/Client.client.luau" }
          }
        }
      }
    }
    ```

    Notice how files with `server` in their name (e.g. `Server.luau`) are routed to `ServerScriptService`, while `Client.luau` and `Utils.luau` go to `ReplicatedStorage.Shared`.
  </Step>

  <Step title="Start Rojo">
    With `default.project.json` in place, start the Rojo dev server to sync your project into Roblox Studio.

    ```bash
    rojo serve
    ```

    Then open Roblox Studio, install the Rojo Studio plugin from the Roblox plugin marketplace, and connect to the running server.
  </Step>

</Steps>

<Tip>
  During active development, use watch mode instead of running `npm run build:rojo` manually. It regenerates `default.project.json` automatically every time you add, remove, or rename a file under `src/`.

  ```bash
  npm run watch:rojo
  ```

  This runs `chokidar "src/**/*" -c "npm run build:rojo"`, so Rojo always has an up-to-date project file without any extra steps.
</Tip>
